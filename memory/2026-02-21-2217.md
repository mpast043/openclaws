assistant: ## Spawning a Sub-Agent (`sessions_spawn`)

### Required Parameters

| Parameter | Required? | Description |
|-----------|-----------|-------------|
| `task` | **Yes** | The task description/instructions for the sub-agent |
| `label` | No | Optional label for logs/UI |
| `agentId` | No | Optional ‚Äî spawn under a different agent (requires allowlist) |
| `model` | No | Optional ‚Äî overrides sub-agent's default model |
| `runTimeoutSeconds` | No | Abort after N seconds (default: infinite) |
| `cleanup` | No | `delete` or `keep` (default: keep) |

---

### Simple Example

```python
# Spawn a sub-agent to do work in isolation
sessions_spawn(
    task="Summarize this document and return key findings",
    label="doc-summarizer",
    runTimeoutSeconds=300  # Kill after 5 min
)
```

**Returns immediately:**
```json
{
  "status": "accepted",
  "runId": "abc-123",
  "childSessionKey": "agent:main:subagent:uuid-here"
}
```

---

### What Happens

1. **New isolated session** created: `agent:main:subagent:<uuid>`
2. **Sub-agent runs** with your task in full isolation
3. **Result announced** back to your channel when complete
4. **Auto-archived** after 60 min (configurable)

---

### Key Limitations

| Restriction | Details |
|-------------|---------|
| **No session tools** | Sub-agents can't use `sessions_list`, `sessions_spawn`, etc. |
| **Non-blocking** | Returns immediately; use `sessions_history` to check progress |
| **No cross-spawn** | Sub-agents can't spawn other sub-agents |
| **Tool set** | Can be restricted via `tools.subagents.tools` config |
| **Allowlist** | Spawning other agent IDs requires `subagents.allowAgents: ["*"]` |

---

### Discover Allowed Agents

```python
agents_list()  # Shows which agent IDs you can spawn under
```

---

### When to Use

‚úÖ Long-running tasks (research, code analysis, data processing)  
‚úÖ Parallel work (spawn multiple for concurrency)  
‚úÖ Isolated experiments (sub-agent fails ‚â† main session fails)  
‚úÖ Delegation (offload to specialized agent personas)

‚ùå Short synchronous tasks (overhead not worth it)  
‚ùå Tasks requiring session tool access (sub-agents blocked)
